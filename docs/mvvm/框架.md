## 框架

为什么会有框架?

自动维护UI和状态同步.

不用操作DOM元素

使用框架的好处? 

1. 组件化
2. 分层次
3. 生态好
4. 开发效率高



### MVVM

Model-View-ViewModel 

设计思想： 关注Model的变化，让MVVM框架去自动更新DOM的状态，从而把开发者从操作DOM的繁琐步骤中解脱出来

Model :  数据

View:  视图

ViewModel: 负责把Model的数据同步到View显示出来，还负责把View的修改同步回Model
